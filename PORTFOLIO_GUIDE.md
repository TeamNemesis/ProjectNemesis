## 2. 단계별 포트폴리오 제작 가이드

### Step 1: 프로젝트 하이라이트 정리 (1-2일)

**목표**: 가장 인상적인 부분을 명확히 파악

#### 기술적 강점 정리
```markdown
1. State Machine Pattern을 활용한 플레이어 상태 관리
   - 복잡한 상태 전환을 명확하게 구현
   - 유지보수 및 확장이 용이한 구조

2. Object Pooling으로 30% 성능 향상
   - 가비지 컬렉션 최소화
   - 안정적인 60 FPS 유지

3. MVC 패턴 적용한 플레이어 시스템
   - 코드 분리 및 재사용성 증대
   - 테스트 및 디버깅 용이
```

#### 개인 기여도 정리
각 팀원이 자신의 역할을 명확히 정리:

**endsun1234 (예시)**:
- 상호작용 시스템 설계 및 구현
- 10개 이상의 상호작용 오브젝트 개발
- 인터페이스 기반 확장 가능한 구조 설계

### Step 2: 시각적 자료 준비 (3-5일)

#### 게임 플레이 영상 제작

**촬영 시나리오**:
```
1. 오프닝 (10초)
   - 게임 타이틀
   - 프로젝트 정보

2. 인트로 (20초)
   - 게임 시작 화면
   - 기본 UI 소개

3. 기본 플레이 (2분)
   - 이동 및 조작
   - 전투 시스템
   - 몬스터 AI

4. 핵심 기능 (2분)
   - 스킬 시스템
   - 무기 전환
   - 상호작용 오브젝트

5. 하이라이트 (1분)
   - 보스 전투
   - 화려한 이펙트
   - 스킬 콤보

6. 엔딩 (10초)
   - 팀 크레딧
   - 연락처/링크
```

**촬영 팁**:
- Unity의 Recorder 패키지 사용
- 1920x1080 이상 해상도
- 60 FPS로 녹화
- 배경 음악 추가
- 자막 추가 (주요 기능 설명)

#### 스크린샷 촬영 가이드

**필수 스크린샷**:
1. 메인 화면 (1장)
2. 전투 장면 (3-4장)
3. 스킬 사용 장면 (2-3장)
4. UI 화면 (2장)
5. 보스 전투 (2장)
6. 맵 구조 (2장)

- 워터마크 추가 (선택사항)

### 예상 기술 질문

#### Unity 관련
```
Q: State Machine Pattern을 적용한 이유는?
A: 플레이어의 여러 상태(Idle, Move, Attack 등)를 명확하게 
   관리하고, 상태 전환 로직을 단순화하기 위해 적용했습니다.
   이를 통해 버그를 90% 줄이고 유지보수성을 크게 향상시켰습니다.

Q: Object Pooling은 어떻게 구현했나?
A: Dictionary로 오브젝트 타입별 Queue를 관리하고, Get/Release 
   메서드로 재사용했습니다. 이로 인해 가비지 컬렉션이 줄어들어
   프레임 드랍이 30% 감소했습니다.

Q: 성능 최적화는 어떻게 진행했나?
A: Unity Profiler로 병목 지점을 찾아 Object Pooling, Addressables,
   NavMesh 최적화를 적용했습니다. 60 FPS를 안정적으로 유지하는
   결과를 얻었습니다.
```

#### C# 관련
```
Q: 인터페이스를 사용한 이유는?
A: 상호작용 오브젝트들이 공통 기능(Interact)을 가지지만 
   구현이 다르기 때문에 IInteractable 인터페이스로 추상화했습니다.
   이를 통해 새로운 오브젝트 추가가 용이해졌습니다.

Q: 싱글톤 패턴의 단점과 대안은?
A: 테스트가 어렵고 전역 상태로 인한 결합도가 높아집니다.
   대안으로는 의존성 주입(DI)을 고려할 수 있지만, 
   이 프로젝트에서는 빠른 프로토타이핑을 위해 싱글톤을 선택했습니다.
```
